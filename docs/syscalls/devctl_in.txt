devctl is a syscall that exposes a driver's interface to userspace.
This is like devctl but can read data from a driver.

Arguments:

RBX: Driver class.
RCX: Request.

Returns:
RAX will hold the data, the only downside to this syscall
is you get no status as of now.


See status in devctl_status.txt


------------------------------------------------------------------------------------------------------------------------------
DRIVER CLASS ID   REQUEST ID           DESCRIPTION                                  TITLE
------------------------------------------------------------------------------------------------------------------------------
0                 0xdeb5a40523c      Sends a request to read                KEYBOARD DRIVER (READ KEYSTROKE)
                                     a keystroke.
